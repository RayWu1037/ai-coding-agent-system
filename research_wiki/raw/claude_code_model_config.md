# Claude Code Model Configuration

The Claude Code model configuration documentation explains how model aliases such as sonnet, opus, and haiku work, and how those settings can be controlled through startup flags, environment variables, and project settings. It also explains that model choice affects planning and execution behavior, not just naming.

This source is useful for our project because early instability came from outdated or implicit model defaults. The documentation supports our later decision to make model configuration explicit and to treat model choice as part of engineering reliability.

Official source: https://code.claude.com/docs/en/model-config
