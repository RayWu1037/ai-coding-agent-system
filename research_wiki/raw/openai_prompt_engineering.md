# OpenAI Prompt Engineering Guide

The OpenAI prompt engineering guide explains that better results come from clear instructions, explicit output expectations, and iterative refinement. It recommends placing instructions early in the prompt, being specific about the desired format, and treating prompting as a process of controlled improvement rather than a one-shot request.

This source is useful for our project because it supports the claim that prompt quality behaves like specification quality. It also supports our observation that narrower prompts with clear constraints reduce token waste and reduce avoidable model errors.

Official source: https://help.openai.com/en/articles/6654000-prompt-engineering-guide
